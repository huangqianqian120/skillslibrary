## 一、项目背景

### 1.1 为什么要做这个？

**现状问题：**

- 重复造轮子：每个 AI Agent 项目都要从头写提示词
- 分散零碎：好用的技能散落在各处，没有统一管理
- 门槛高：新手不知道什么叫"好的 Agent 技能"
- 复用难：自己写的技能很难分享给其他人

---

## 二、产品定义

**一句话定义：**

> AI Agent 技能库 —— 让每个 AI Agent 都能快速拥有 50+ 种能力

## 四、技术架构

### 4.1 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | Next.js 14 + TypeScript + Tailwind CSS |
| 部署 | Vercel |
| 数据 | GitHub |
| 统计 | 自建 API + 日志 |

### 4.2 项目结构

```
skillslibrary/
├── app/                    # Next.js 应用
│   ├── api/visit/        # 访问统计 API
│   ├── skill/[id]/      # 技能详情页
│   └── page.tsx          # 首页（含分页+CLI）
├── components/            # 组件
│   ├── CLISection.tsx   # CLI 教程组件
│   └── VisitTracker.tsx  # 访问追踪组件
├── cli/                   # CLI 工具
│   ├── package.json
│   └── bin/skills.js
├── skills/               # 技能数据
└── logs/                # 访问日志
```
