# 用户洞察分析器

对用户研究数据进行深度分析，提取可行动的洞察。

## 功能

- 🔍 **主题提取** - 自动识别关键主题和模式
- 💡 **洞察发现** - 发现深层需求、痛点、机会点
- 🎯 **优先级排序** - 按影响力和可行性排序
- 📊 **可视化输出** - 生成分析图表和报告
- 🔗 **关联发现** - 发现隐藏的因果关系

## 使用方法

### 分析数据

```
分析
来源：[访谈记录/问卷/评论/反馈]
类型：[探索性 / 验证性]
重点关注：[用户需求/体验痛点/竞品对比]
```

### 洞察类型

| 洞察类别 | 说明 | 示例 |
|----------|------|------|
| 需求洞察 | 用户真正想要什么 | "用户想要的不是快，而是确定性" |
| 痛点洞察 | 用户卡在哪里 | "第一步太难，流失率高" |
| 机会洞察 | 未被满足的市场空白 | "30-40岁女性市场空缺" |
| 行为洞察 | 用户实际做了什么 | "嘴上说环保，行动不支持" |
| 情感洞察 | 用户感受如何 | "焦虑来自信息过载" |

## 分析维度

```
1. 认知层面
   - 意识程度（无意识到有意识）
   - 知识水平（新手到专家）
   - 决策信心

2. 行为层面  
   - 行动频率
   - 路径模式
   - 障碍识别

3. 情感层面
   - 满意度
   - 推荐意愿
   - 情感驱动

4. 价值层面
   - 付费意愿
   - 价值认知
   - 替代方案
```

## 示例

```
用户：分析
来源：20份智能门锁访谈记录
类型：探索性
重点关注：购买决策因素

AI：好的，正在分析...

## 用户洞察报告

### 执行摘要
分析20份访谈，发现3个核心洞察

### 🎯 一级洞察

#### 洞察 1：安全焦虑是核心驱动力
**洞察内容**：用户购买智能锁的最大动机不是方便，
而是"担心忘记锁门"的安全焦虑。

**证据**：
- 18/20 用户提到"万一忘记锁"
- "安全感"出现频次：23次
- "方便"仅出现：8次

**置信度**：高

**建议**：
- 营销强调"安心"而非"便捷"
- 功能优先级：自动上锁 > 远程查看

#### 洞察 2：安装服务是决胜点
**洞察内容**：用户对安装服务的担忧超过产品本身，
10个用户提到"怕装不好"。

...
```

## 配置文件

```yaml
analysis:
  min_evidence: 3        # 最少证据数
  confidence_threshold: 0.6  # 置信度阈值
  max_insights: 10       # 最大洞察数
  
output:
  format: markdown       # 输出格式
  include_charts: true  # 包含图表
  include_quotes: true   # 包含用户引用
```
