# 用户研究框架

基于专家方法论的研究设计工具。

## 功能

- 📐 **框架生成** - 根据研究目标自动推荐研究框架
- 📝 **提纲设计** - 自动生成访谈/调研提纲
- 🎯 **方法指导** - 推荐最佳研究方法
- ✅ **质量检查** - 研究设计合理性评估
- 📚 **知识库** - 集成经典研究方法论

## 内置方法论

| 方法论 | 适用场景 | 特点 |
|--------|----------|------|
| 5 Why 分析 | 挖掘深层原因 | 连续追问5次为什么 |
| 旅程地图 | 体验优化 | 梳理用户完整旅程 |
| 人物志 | 用户理解 | 创建典型用户画像 |
| 卡片分类 | 信息架构 | 理解用户心智模型 |
| 情境探询 | 需求发现 | 在真实场景中观察 |
| 行为三角 | 动机分析 | 行为-情感-情境关联 |

## 使用方法

### 创建设计

```
设计研究
目标：[研究目标，如：优化注册流程]
阶段：[探索发现 / 验证假设 / 深度诊断]
团队：[独立/协作]
```

### 框架生成

根据目标推荐框架：

```
研究目标              推荐方法           优先级
─────────────────────────────────────────────
发现用户需求          深度访谈+情境观察    ⭐⭐⭐
验证功能假设          用户测试+A/B测试    ⭐⭐⭐
优化体验流程          旅程地图+可用性测试  ⭐⭐
理解市场机会          竞品分析+用户调研    ⭐⭐
诊断流失原因          访谈+行为数据分析    ⭐⭐⭐
```

### 提纲生成

自动生成访谈/调研提纲：

```
生成提纲
类型：[深度访谈/问卷/用户测试]
主题：[具体话题]
时长：[预估分钟数]
深度：[表层/中等/深入]
```

输出结构化提纲，包含：
- 开场话术
- 核心问题（按模块组织）
- 追问提示
- 结束语

## 研究设计模板

### 探索性研究（1-2周）

```
阶段1：文献调研（2-3天）
- 竞品分析
- 行业报告
- 现有数据

阶段2：用户访谈（5-7天）
- 8-12个深度访谈
- 30-45分钟/场

阶段3：分析综合（3-5天）
- 主题分析
- 洞察提炼
```

### 验证性研究（1周）

```
- 假设明确
- 快速测试（5-8人）
- 数据分析
- 结论输出
```

## 质量评估

研究设计检查清单：

```
✅ 研究目标清晰可衡量
✅ 样本选择有代表性
✅ 方法与目标匹配
✅ 问题设计无引导性
✅ 访谈大纲逻辑通顺
✅ 时间预算合理
✅ 预设分析计划
✅ 伦理审查通过
```

## 输出示例

```markdown
## 用户研究设计方案

### 研究概览
- 目标：优化电商App结账流程
- 方法：可用性测试 + 访谈
- 周期：2周
- 样本：8人

### 研究框架

1. 开场（5min）
   - 破冰
   - 说明流程
   - 获取同意

2. 任务测试（20min）
   - 任务1：完成购买
   - 任务2：使用优惠券
   - 任务3：查看物流

3. 深度访谈（15min）
   - 体验感受
   - 痛点挖掘
   - 期望对比

### 访谈提纲

开场：
"嗨，感谢你今天来！今天主要想请你帮忙测试一下
我们的App，不用担心，我们是在测试产品，不是你～
大概需要30分钟，有什么问题随时问我。"

核心问题：
1. 平时网购多吗？主要买什么？
2. 结账时你最看重什么？
3. 有没有遇到过什么问题？
...
```

## 与其他技能配合

|--------|----------|
| 完整研究 | 框架 → 访谈 → 分析 → 画像 |
| 快速洞察 | 框架（简化）→ 快速访谈 → 洞察 |
| 持续优化 | 框架 → 定期访谈 → 对比分析 |
